# Opticrew.io — AI-Powered Programmatic SEO & Autopilot Content Engine
## Implementation Plan

A full-stack multi-tenant SaaS platform where agencies and end-users generate, publish, and manage hundreds of SEO articles on autopilot using a crew of AI agents — all branded under their own domain.

---

## 🏗️ Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| **Framework** | Next.js 16 (App Router) | Server Components + Node.js API Routes for the Backend |
| **Language** | TypeScript | Type safety across the whole stack |
| **Styling** | Tailwind CSS v4 | Already installed in project |
| **Database** | MySQL on AWS (RDS/EC2) | Highly scalable, reliable backend storage |
| **ORM** | Prisma | Type-safe DB queries, easy migrations |
| **Auth** | NextAuth.js v5 (Auth.js) | Multi-role: SUPER_ADMIN, AGENCY_OWNER, CLIENT |
| **AI — Content** | OpenAI GPT-4o API & Gemini | Article generation with strict prompt engineering |
| **AI — Filtering** | Anthropic Claude API | Keyword intent classification |
| **Search Grounding** | Tavily AI Search API | Real-time web facts to prevent hallucination |
| **Keyword Research** | DataForSEO API | Authentic search volumes, long-tail keywords |
| **CMS Publisher** | WordPress REST API + Webflow CMS API | One-click publish (Shopify in Phase 2) |
| **Images** | Unsplash API | Free featured images for articles |
| **Background Jobs** | Vercel Cron Jobs | Async article generation queue |
| **Payments** | PayPal API | Monthly plans + LTD package + credit top-ups in USD |
| **Email** | Resend | Transactional emails |
| **File Storage** | Cloudflare R2 / AWS S3 | Logo, favicon uploads for white-label tenants |

## 🗄️ Database Schema (Prisma / MySQL)

### Core Tables

```
tenants          → Agency white-label configs (domain, branding, credits)
users            → All roles (SUPER_ADMIN | AGENCY_OWNER | CLIENT)
integrations     → WordPress/Webflow API keys per tenant
seo_campaigns    → Bulk generation campaigns
articles         → Individual generated articles
keyword_clusters → Keywords grouped by intent per campaign
subscriptions    → PayPal plan data per user/tenant
credit_logs      → Audit trail of AI credit consumption
```

### Key Relationships
- `tenants` ← `users` (tenant_id FK) — Data isolation per agency
- `tenants` ← `integrations` — Each agency connects their own CMS
- `seo_campaigns` ← `keyword_clusters` ← `articles` — Cascade flow
- `tenants.ai_credit_balance` — Decremented on every article generation

## ⚙️ Core Engines — Implementation Detail

### Engine 1: AI Keyword Mapper
**Route:** `POST /api/keywords` (Node.js)

1. User submits seed keyword + target country
2. Call **DataForSEO** Keyword Ideas API → returns 200–500 raw keywords with search volume
3. Pass keyword list to **Claude API** — prompt asks to classify by intent (Informational / Commercial / Transactional) and group into clusters
4. Store clusters in `keyword_clusters` table linked to campaign
5. UI shows interactive table with search vol, intent badge, competitor difficulty score

### Engine 2: AI Content Autopilot
**Route:** `POST /api/articles/generate` (background job)

1. Retrieve keyword cluster from DB
2. For each keyword in batch:
   a. Call **Tavily Search API** → get top 3 real-time search result snippets (facts, dates, sources)
   b. Build prompt utilizing **GPT-4o / Gemini** with: keyword, intent type, Tavily context, strict instructions (Markdown format, H2/H3 structure, no fluff, natural keyword placement)
   c. Stream response or store full completion
3. Decrement `tenants.ai_credit_balance` after each successful generation
4. If balance = 0 → mark campaign as QUOTA_EXHAUSTED, block further generation
5. Article stored with status `DRAFT`

### Engine 3: One-Click CMS Publisher
**Route:** `POST /api/publish` (Node.js)

1. User selects articles (or all DRAFT in campaign)
2. Fetch integration credentials from `integrations` table (decrypted)
3. Fetch featured image from **Unsplash API** based on article topic
4. Branch on `cms_type`:
   - **WordPress:** `POST /wp-json/wp/v2/posts` with title, content (convert MD→HTML), featured_media
   - **Webflow:** Webflow CMS Items API with field mapping
5. Mark article as `PUBLISHED`, store `published_url`
6. Optional: schedule publish via date picker → cron job handles delayed publish

## 🌐 Multi-Tenant Middleware (White-Label Core)

```typescript
// middleware.ts
// 1. Read request host header
// 2. Query DB: SELECT * FROM tenants WHERE custom_domain = host
// 3. If match → inject tenant context into request headers
// 4. Next.js layout reads tenant data → renders custom logo, colors
// 5. No tenant found → render default Opticrew.io branding
```

The white-label UI theming is driven by CSS custom properties injected per-tenant:
```css
--brand-primary: {tenant.brand_color_hex};
--brand-logo: url({tenant.logo_url});
```
